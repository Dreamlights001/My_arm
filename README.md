# 3kg 双臂分拣机械臂（单臂）方案报告

> 更新日期：2026-04-09  
> 版本说明：本版将之前分散内容重组为报告结构，并保留关键历史表格（目标力矩对比表、L3 扫描表、精度计算过程）。

## 1. 执行摘要

本项目目标是设计一套 `6DoF + 1 夹爪` 的低成本机械臂（双臂同构，先设计单臂），用于传送带瑕疵分拣，并兼容 `ROS2 + VLA`。  
在当前参数下（`L2=350 mm`，`L3=250 mm`，中置 J4 偏转电机）：

- `3kg` 负载工况下，关键关节 `J2/J3/J4` 扭矩裕量分别约 `35.5% / 45.3% / 38.1%`。
- 基于官方分辨率参数，当前控制量化误差 `RSS≈0.380 mm`，并入杆件静态挠度后约 `0.388 mm`。
- 工程重复定位仍预计 `2-3 mm`，主因是回差/装配/标定误差，不是编码分辨率本身。
- 若目标 `0.5 mm`，非量化误差预算需压到 `<=0.315 mm`（优先治理 J2/J3 传动回差与输出轴反馈）。

## 2. 项目背景与输入条件

## 2.1 任务场景

- 应用：移动产线瑕疵分拣（仿真/设计阶段，不做实物）。
- 双臂系统同侧部署，两臂同构。
- 传送带宽度：`0.2 m`。
- 工件不堆叠、线性排列。

## 2.2 设计约束与目标

- 末端额定抓取：`3 kg`。
- 机械结构：`6DoF + 1 末端执行器`。
- 控制兼容：`ROS2`，上层适配主流 `VLA`（关节空间动作）。
- 成本导向：优先 OCS + FEETECH 方案。

## 2.3 参数假设说明

- 你最初给出的体积参数存在冲突（`0.5 dm^3` vs `0.05 m^3`）。本报告按可实现分拣场景采用前者量级。
- 断电阻尼力矩多数型号无官方公开值，报告中将“官方值”和“工程估算值”严格分开。

## 3. 机械总体方案

## 3.1 连杆拓扑（6DoF 对应 5 主连杆）

- `L1 = 180 mm`：基座安装面到 J2 轴心垂直高度。
- `L2 = 350 mm`：大臂（J2 -> J3）。
- `L3a = 150 mm`：小臂前段（J3 -> J4中置节点）。
- `L3b = 100 mm`：小臂后段（J4节点 -> J5）。
- `L4a = 55 mm`：腕段（J5 -> J6）。
- `L4b = 35 mm`：腕末段（J6 -> 工具法兰）。
- `Ltool = 100 mm`：工具法兰到 TCP。

水平最大前伸半径：

`R = 350 + 150 + 100 + 55 + 35 + 100 = 790 mm`

## 3.2 建议工作空间（单臂）

- 沿线 `X`：约 `0.60 m`
- 横向 `Y`：约 `0.30 m`（覆盖 0.2m 传送带并留边）
- 高度 `Z`：约 `0.35 m`

## 3.3 中置偏转电机（J4）布置

- J4 电机放在 `L3a/L3b` 分界处，短皮带驱动 J4 轴。
- 相比把 J4 电机放在腕部末端，肩/肘设计扭矩可下降约 `1-2%`，远端惯量更小。
- 对 CAD 更友好：只增加 1 套中置支架与短轴承座。

## 7. 精度分析与 0.5mm 路线

## 7.1 基于官方参数的控制量化精度（可复现计算）

计算定义：

- 舵机侧角分辨率：`DeltaTheta_servo`
- 关节输出角分辨率：`DeltaTheta_out = DeltaTheta_servo / r`（`r` 为外部减速比）
- TCP 单关节量化误差（1 LSB）：`e_i(step) = l_i * rad(DeltaTheta_out)`
- TCP 单关节量化误差（±0.5 LSB）：`e_i(±0.5) = 0.5 * e_i(step)`
- RSS 合成：`e_RSS = sqrt(sum(e_i^2))`

示例（J2）：

- `DeltaTheta_out = 0.1 / 2.5 = 0.04 deg`
- `e2(step) = 0.79 * rad(0.04 deg) = 0.000551524 m = 0.551524 mm`
- `e2(±0.5LSB) = 0.275762 mm`

几何力臂（当前结构）：

- `l2 = 0.79 m`
- `l3 = 0.44 m`
- `l4 = 0.19 m`
- `l5 = 0.10 m`
- `l6 = 0.05 m`

最终精度汇总表（基于官方参数）：

| 关节 | 舵机侧分辨率来源 | `DeltaTheta_servo` (deg) | 减速比 `r` | 关节输出分辨率 `DeltaTheta_out` (deg) | 力臂 `l_i` (m) | `e_i(step)` (mm) | `e_i(±0.5LSB)` (mm) |
|---|---|---:|---:|---:|---:|---:|---:|
| J2 | OCS CAN 协议位置设定精度 | `0.100000` | `2.5` | `0.040000` | `0.79` | `0.551524` | `0.275762` |
| J3 | OCS CAN 协议位置设定精度 | `0.100000` | `1.7` | `0.058824` | `0.44` | `0.451732` | `0.225866` |
| J4 | OCS CAN 协议位置设定精度 | `0.100000` | `1.6` | `0.062500` | `0.19` | `0.207258` | `0.103629` |
| J5 | FEETECH 12bit/4096 | `0.087891` | `1.0` | `0.087891` | `0.10` | `0.153398` | `0.076699` |
| J6 | FEETECH 12bit/4096 | `0.087891` | `1.5` | `0.058594` | `0.05` | `0.051133` | `0.025566` |
| 合成（RSS） | 仅控制量化 | - | - | - | - | `0.759830` | `0.379915` |
| 合成（RSS + 梁挠度0.08mm） | 控制量化 + 结构静态挠度 | - | - | - | - | - | `0.388247` |

关键结果：

- 若按“1 LSB 最坏步距”估计：`0.760 mm`。
- 若按“量化误差 ±0.5 LSB（更符合统计实际）”估计：`0.380 mm`。
- 并入杆件静态挠度 `0.08 mm` 后：`0.388 mm`。

## 7.2 达到 0.5mm 的必要条件（当前方案下）

对总误差目标 `0.5 mm`，已知“量化 + 杆件静态挠度”约 `0.388 mm`，则其余误差（回差、装配、标定、温漂、控制模型）预算上限：

`e_other_max = sqrt(0.5^2 - 0.388247^2) = 0.315063 mm`

优先级（按影响量和实施成本）：

1. J2/J3 先治理（量化贡献最大，且回差放大最明显）：输出轴编码器闭环 + 低回差传动。  
2. J2/J3 轴承预紧与同轴度控制：降低反向死区与负载变形。  
3. J4 次级优化：缩短传动链、提高中置节点刚度。  
4. 控制补偿：重力/摩擦/死区补偿 + 低速 S 曲线，压低剩余误差。

## 8.3 带传动 vs 齿轮传动：对精度的影响

误差映射公式（统一口径）：

- `e_tcp = l * rad(DeltaTheta)`
- 本节示例按 `l = 0.79 m`（J2 到 TCP 代表力臂）换算。

回差/迟滞引起的等效 TCP 误差（换向场景）：

| 传动方案 | 典型等效角误差（输出端） | 换算后 `e_tcp`（mm） | 主要误差机理 |
|---|---:|---:|---|
| 预紧同步带（HTD/GT） | `0.03° ~ 0.08°` | `0.414 ~ 1.103` | 皮带弹性 + 齿形啮合迟滞 |
| 普通单级齿轮（有侧隙） | `0.05° ~ 0.20°` | `0.689 ~ 2.758` | 齿侧间隙导致反向死区 |
| 预紧消隙齿轮副 | `0.01° ~ 0.03°` | `0.138 ~ 0.414` | 预紧后剩余微小弹性与制造误差 |

载荷弹性引起的角偏差（同样影响定位）：

- `DeltaTheta_load = T / k_theta`
- 以 `T = 20 N.m` 为代表工况估算：

| 传动方案 | 扭转刚度 `k_theta`（工程典型） | `DeltaTheta_load`（deg） | 换算后 `e_tcp`（mm） |
|---|---:|---:|---:|
| 同步带一级 | `1.5e4 ~ 3.0e4 N.m/rad` | `0.076 ~ 0.038` | `1.047 ~ 0.524` |
| 钢齿轮一级 | `8.0e4 ~ 2.0e5 N.m/rad` | `0.014 ~ 0.006` | `0.193 ~ 0.077` |

结论（面向本项目）：

- 带传动优势是结构简单、成本低、噪声小，且通过预紧可把“几何回差”压得较小。
- 齿轮传动优势是刚度高、载荷弹性误差小；若做预紧消隙，换向精度通常优于普通带传动。
- 你当前 3kg 分拣方案若目标维持在 `1~2 mm`，预紧同步带可用；若要稳定 `<=0.5 mm`，J1/J2/J3 更建议用低回差齿轮方案（预紧齿轮/谐波/低回差行星）并叠加输出轴编码器闭环。

## 9. 断电阻尼力矩（Power-off Backdrive）

## 9.1 官方数据状态

- 当前选型的 OCS/FEETECH 产品页多数未公开“断电阻尼力矩/反拖力矩”。
- 设计上必须按“断电无保持”处理，不依赖自锁。

## 9.2 工程估算（仅仿真初值）

估算方法：按额定扭矩 `3%-8%` 估算断电反拖阻尼。

| 型号 | 额定扭矩 (N.m) | 估算断电阻尼力矩 (N.m) |
|---|---:|---:|
| OCS-D9501 | 34.32 | 1.03 - 2.75 |
| OCS-D6501 | 24.52 | 0.74 - 1.96 |
| OCS-D2002 | 8.34 | 0.25 - 0.67 |
| SM-260B-C001 | 6.37（按保守 `65 kg.cm`） | 0.19 - 0.51 |
| STS3250 | 1.57 | 0.05 - 0.13 |
| STS3215 | 0.98 | 0.03 - 0.08 |

## 9.3 实测流程

- 输出轴加力臂 `r`（建议 `0.10 m`）。
- 缓慢施加拉力 `F`，记录起转临界值。
- `T_d = F * r`，正反向各 5 次，取均值和区间。

## 10. ROS2 与 VLA 适配

- 单臂动作向量：`[j1, j2, j3, j4, j5, j6, gripper]`。
- 控制框架：`ros2_control + joint_trajectory_controller`。
- 硬件层：OCS/FEETECH 统一通过 RS485/Modbus 适配为关节接口。
- 双臂方案：左右臂动作拼接并做时间同步。

## 12. 实施与验证计划

1. 先做单臂关键台架：J2/J3/J4/J6+夹爪。  
2. 验证三项：连续温升、3kg 轨迹误差、重复抓放成功率。  
3. 若重复定位 >3 mm：优先改 J2/J3 反馈与传动回差，再做控制补偿。  
4. 若目标升到 0.5 mm：J2/J3 进入高精关节方案（低回差传动 + 输出编码器闭环）。
